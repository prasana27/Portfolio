Accessibility Audit

Producing an audit report to identify if site is accessible to all users that want medical advice

Overview

The aim of the audit report was to find if the site Evidently Cochrane (www.evidentlycochrane.com) conforms to level AA based on WCAG 2.1 guidelines and to provide solutions if they don't.

Executive Summary

  • The site does not conform to level AA based on the WCAG 2.1 guidelines
  • There are severe accessibility issues affecting visually impaired users and motor-impaired users
  • Potential users are losing out on reading reviews of medical research due to poor site design
  • Site would need to invest in resolving issues, but most problems can be solved through minor tweaks
  • A user feedback option should be implemented sitewide so that users can report issues
  • The site should undergo user testing with people with varying levels of disabilities
  • Site should undergo another audit after tweaks and user testing to check for level AA conformance

Technology used for audit

  • Used the environment of Macbook Pro running macOS Mojave on Google Chrome version 78
  • Chrome's inbuilt 'Inspect' tool was used to check the HTML and CSS code
  • Installed a browser extension 'WAVE Evaluation Tool' to identify WCAG 2.1 issues on the site page
  • Used Apple's inbuilt screen reader 'VoiceOver' to see how visually-impaired users will use the site
  • Keyboard focus was used to check if motor-impaired can operate site using just keyboard
  • Filters such as grayscale were used to check if colour blind users can operate site
